# Allposty

**Vision:** A SaaS social media scheduling and management platform that lets creators, agencies, and SMBs plan, compose, schedule, and publish content across all major social networks from a single dashboard — with AI-assisted caption writing.

**For:** Creators, agencies, and SMBs managing social media across multiple accounts and platforms.

**Solves:** The cost and complexity of tools like Buffer, Hootsuite, or Sendible — allposty provides the same core capabilities without per-seat or per-channel pricing traps, with AI built in.

---

## Goals

- **G1:** First paying customer within 3 months of backend work starting.
- **G2:** Support the 6 core platforms (Instagram, Facebook, TikTok, YouTube, Twitter/X, LinkedIn) with publish + schedule in v1.
- **G3:** AI caption generation converts free → paid at >20% rate.
- **G4:** Public REST API + SDK from day 1 — allposty's own frontend is just a client.

---

## Tech Stack

**Frontend:**
- Framework: React 19 + Vite 7 (TanStack Router, TanStack Query)
- Language: TypeScript
- UI: Radix UI + Tailwind CSS v4 + shadcn components
- Auth: better-auth
- API client: hey-api (generated from OpenAPI spec)
- Hosting: Cloudflare Pages

**Backend:**
- Framework: Go + Fiber v2
- Language: Go 1.22+
- Database: PostgreSQL (Neon or Railway Postgres)
- Cache / Job Queue: Redis + Asynq (background job scheduler)
- Media Storage: Cloudflare R2 (S3-compatible)
- Auth: JWT (issued by backend, validated by Fiber middleware)
- Billing: Stripe (subscription tiers)
- AI: OpenAI API (GPT-4o for caption generation)
- Hosting: Railway (or Render / Fly.io)

**Key dependencies (backend):**
- `gofiber/fiber/v2` — HTTP framework
- `asynq` — Redis-backed background job queue
- `gorm` + `pgx` — PostgreSQL ORM + driver
- `golang-jwt/jwt` — JWT auth
- `stripe-go` — Billing
- `aws-sdk-go-v2` — R2/S3 media uploads

**Key dependencies (frontend):**
- `@tanstack/react-router` + `@tanstack/react-query`
- `better-auth` — frontend auth client
- `@hey-api/openapi-ts` — generated API client from backend OpenAPI spec
- `recharts` — analytics charts
- `framer-motion` — animations

## Scope

**v1 includes:**
- Multi-tenant: organizations → workspaces → members (RBAC)
- Social account OAuth connections: Instagram, Facebook, TikTok, YouTube, Twitter/X, LinkedIn
- Post composer with per-platform caption + media overrides
- Scheduling engine (Asynq-based background jobs)
- Visual content calendar
- AI caption/description generation (OpenAI GPT-4o)
- Media library (upload, organize, reuse)
- Subscription billing (Stripe, 3 tiers)
- Public REST API with versioning (`/api/v1/...`)
- OpenAPI spec (served by backend, consumed by frontend codegen)

**Explicitly out of scope for v1:**
- AI image generation (v2)
- Social inbox / unified DMs + comments (v2)
- Approval workflows / client portal (v2)
- Bluesky, Threads, Pinterest, Mastodon support (v2 — design provider interface to make these easy to add)
- Mobile app
- White-label customization (Agency tier feature, v2)
- N8N / Make.com integrations
- Analytics beyond basic post performance (v2)

---

## Constraints

- **Architecture:** Separate repos; API-first (OpenAPI spec is source of truth)
- **Providers:** OAuth flows written from scratch in Go — no code copy from brightbean (patterns/logic only, different language)
- **Billing:** Must be live before public launch (no free unlimited tier exploit)
- **Data isolation:** Each organization's data must be strictly isolated at DB query level
